# Workflow - Decision

1. Clarify the decision to be made.
2. Read existing `docs/decisions/`, `docs/architecture/`, `docs/context/`, and `docs/memory/`.
3. Compare realistic options with pros and cons.
4. Create an ADR from `docs/decisions/_template.md`.
5. Update affected `docs/architecture/` and `docs/context/` files.
6. Update memory if the decision creates a reusable rule.
