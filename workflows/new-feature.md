# Workflow - New Feature

1. Read relevant files in `memory/`, `decisions/`, `architecture/`, and `context/`.
2. Create or update a feature spec in `features/` from `_template.md`.
3. Record non-obvious decisions in `decisions/`.
4. Update context or architecture docs for impacted areas.
5. Generate prompts in `prompts/` when implementation should be split into steps.
6. Implement only when the user explicitly asks for code changes.
7. Verify with available project commands.
8. Record learnings and mistakes in `memory/`.
