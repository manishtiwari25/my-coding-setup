# Product Context

## Product Direction

Amistio is intended to become a SaaS/app brain for software projects: a UI-driven orchestrator that helps users turn product intent into architecture, decisions, feature specs, memory, plans, and implementation prompts.

## Current State

- Product source code has not been added yet.
- The root control plane is active and should be used before implementation.
- The first product decision should choose the initial user surface and execution model.
- This repository is now maintained as a template source; repos created from it should update this file before adopting or changing the product direction.

## Non-Goals Until Decided

- Do not add hosted auth, billing, database, or cloud execution until ADRs define them.
- Do not add product code before the initial MVP scope is written in `features/`.
